\# potent-spec



This repository contains the canonical specification of the Potent system.



A Potent is a boundary that holds form without ever becoming what it contains. This repository defines that boundary as a formal, minimal specification.



\---



\## Canonical Surface



The authoritative definition is:



\- SPECIFICATION.md



This file contains the complete and final structure of the system. It should be read as the source of truth.



\---



\## Repository Contents



\- SPECIFICATION.md — canonical system definition

\- docs/workshop/ — formation path that produced the specification

\- docs/reference/ — source-adjacent materials used during development

\- docs/placement/ — repository posture and usage boundaries



\---



\## What Belongs Here



\- The Potent system definition

\- Supporting materials that preserve how the system was formed

\- Reference materials that informed the specification



\---



\## What Does Not Belong Here



\- Application implementations

\- Runtime code

\- Interpretations, extensions, or redesigns of the system

\- Derived frameworks or tooling



\---



\## Usage



This repository is a specification surface.



It is intended to be:



\- read

\- referenced

\- implemented elsewhere without modification to the core definition



All implementations must treat SPECIFICATION.md as invariant.
