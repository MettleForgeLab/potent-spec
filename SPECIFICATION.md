# Potent System — Minimal Specification



---



## 1. Core

- Single invariant center

- Immutable

- Enforces boundary, validation, refusal, termination

- Identical across all instances



---



## 2. Membrane

- Declared per invocation

- Adaptive in access only

- Cannot alter core, payload meaning, or enforcement rules

- Valid only if aligned with core invariants



---



## 3. Payload

- External, attached input only

- May be present or absent

- Never alters core or membrane identity

- Continuity only via explicit reattachment



---



## 4. Interaction

- Single-shot boundary event

- No hidden state

- No implicit carry-over

- Dissolves after execution



---



## 5. Multi-Attachment

- Partitioned access channels

- No implicit merging

- Relations constructed per interaction only



---



## 6. Continuity

- Forbidden inside core and membrane

- Allowed externally as record

- Allowed in payload only via explicit reattachment



---



## 7. Handoff (Crossing)



Eligible output must include:

- Explicit bounded artifact

- Declared structural form

- Non-inheritance clause



Eligibility is binary, structural, non-interpretive



---



## 8. Composition

- No pipelines

- No shared state

- Sequence = external re-declaration per step

- System scales by resets, not continuity



---



## 9. External Layers

- Orchestrator — non-binding selector

- Record Layer — external memory, non-intrusive

- Profile Registry — optional constraint shapes, non-authoritative



---



## 10. Failure

- Immediate termination

- No in-place recovery

- No residue

- External recording only



---



## 11. Boundary Law

- Payload never touches core

- Membrane mediates but cannot mutate

- Core cannot be modified, extended, or inferred



---



## 12. System Identity



A Potent is:

- A boundary event

- Instantiated by declaration

- Validated before interaction

- Dissolved after execution



---



## ∴ Invariant Principle



Only what is explicitly declared may pass.  

Nothing else continues.
